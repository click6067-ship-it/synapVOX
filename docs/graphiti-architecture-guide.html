01 · 큰 그림

누가 무엇을 하나 — 6개 레이어

핵심은 Graphiti가 DB가 아니라, Neo4j(창고)와 OpenAI(두뇌) 사이에서 그래프를 만들고 검색하는 "조수" 레이어라는 점입니다.

브라우저
React 앱 (react-force-graph)화면·그래프 시각화·질문 입력. 그래프를 캔버스에 그림.
Vercel
프론트엔드 정적 호스팅빌드된 HTML/JS를 CDN으로 배급. 로직은 브라우저에서.
FastAPI
백엔드 API (Render)모든 요청의 창구. 인증·사용량 한도·라우팅. 직접 계산은 아래로 넘김.
Graphiti
지식그래프 프레임워크 ★ 핵심글 → 개념·관계 추출(쓰기) / 하이브리드 검색(읽기). Neo4j·OpenAI를 부린다.
OpenAI
gpt-5.6 + 임베딩 모델실제 언어 이해/생성. 개념 추출·답변 생성·의미 벡터화를 담당하는 두뇌.
Neo4j
그래프 데이터베이스 (Aura)진짜 저장소. 노드(점)·엣지(선)로 데이터가 실제로 쌓이는 창고.

★ 한 줄로: Graphiti = Neo4j(창고) 위에서 OpenAI(두뇌)를 써서 "지식그래프를 채우고 검색"하는 라이브러리.

02 · 워크플로우

일어나는 일은 딱 두 가지

데이터를 넣을 때(쓰기)와 꺼낼 때(읽기=RAG). 이 두 흐름만 이해하면 시스템의 90%가 잡힙니다.

강의 추가 · POST /ingest-text

① 넣기 — 글을 그래프로

  1. 강의 텍스트 투입사용자가 노트 붙여넣기
  2. LLM이 개념·관계 추출Graphiti.add_episode → gpt-5.6
  3. 각 개념을 임베딩(벡터)로OpenAI 임베딩 · 3072차원
  4. 중복 개념 합치기"경사하강법" 두 번 → 한 노드
  5. 노드·엣지로 Neo4j에 저장:Episodic·:Entity · MENTIONS·RELATES_TO
결과: 어지러운 글이 개념 지도가 되어 창고에 쌓임. 사람이 태깅 안 함 — LLM이 자동으로.
질문하기 · GET /ask

② 꺼내기 — 이게 RAG

  1. 질문 입력"역전파가 왜 필요해?"
  2. 하이브리드 검색으로 근거 찾기Graphiti.search — 의미+키워드+그래프
  3. 관련 사실·근거 세션 확보R · Retrieval(찾기)
  4. 근거를 gpt-5.6에 넣어 답 생성G · Generation(답하기)
  5. 답변 + 근거 세션 + 그래프 하이라이트근거 개념에 파란 링
RAG = 찾기(R) + 답하기(G). LLM이 지어내지 않고 내 강의를 근거로 답하는 이유.
곁가지 — "그래프 시각화"는 RAG가 아니다. GET /graph는 검색도 LLM도 안 씀. 창고(Neo4j)에 이미 저장된 개념 지도를 Cypher로 그냥 읽어 그리는 것뿐 — 즉각적이고 AI 비용 0.

03 · 직접 해보기

그래프 실습 — 넣기·꺼내기를 눈으로

노드에 마우스를 올리면 정체(강의·개념·관계)를 알려줍니다. 버튼으로 ① 넣기(그래프가 자라남)와 ② 질문=RAG(근거가 켜짐)를 시연해 보세요.

노드에 마우스를 올려보세요 — 강의·개념·관계가 무엇인지 알려줍니다.

범례

  • 과목 허브 (딥러닝)
  • 강의 · :Episodic
  • 핵심 개념 (여러 강의에 등장)
  • 일반 개념 · :Entity
  • 관계 (MENTIONS / RELATES_TO)
  • 교차연결 (과목 간 공유 개념)

① 넣기 = add_episode가 개념을 뽑아 그래프를 세우는 과정. ② 질문 = search가 근거를 켜고 gpt-5.6이 답하는 RAG.

04 · 개념 사전

핵심 용어의 원리

각 개념 = 무엇인지 한 줄 + 이 프로젝트에서 어떻게 쓰이는지.

그래프 DB Neo4j

데이터를 표(행·열)가 아니라 점(노드)+선(엣지)으로 저장. 관계 탐색이 빠름. 질의어는 Cypher.

여기선 Neo4j Aura(클라우드)에 그래프가 실제 저장.

노드 · 엣지 nodes · edges

노드=사물(강의·개념), 엣지=관계. 엣지는 방향과 "사실(fact)"을 가짐.

여기선 :Episodic·:Entity 노드, MENTIONS·RELATES_TO 엣지.

지식그래프 Knowledge Graph

개념과 관계를 구조화한 그래프. 문서뭉치가 아니라 연결된 지식망이라 관계형 질문에 강함.

여기선 강의에서 뽑은 개념들의 과목별 지식망.

임베딩 embedding

글의 의미를 숫자 벡터로(3072차원). 뜻이 비슷하면 벡터가 가까움 → 단어 달라도 의미로 검색.

여기선 개념마다 임베딩 저장 → 의미 검색 재료.

의미검색 vs 키워드 semantic · BM25

의미검색=임베딩 거리, BM25=단어일치. 강점이 달라 둘 다.

여기선 Graphiti가 둘을 함께 돌림.

하이브리드 검색 hybrid search

의미+BM25+그래프 순회를 합치고 RRF로 재순위. Graphiti 검색의 핵심.

여기선 /ask가 이 검색으로 근거를 모음.

RAG Retrieval-Augmented Generation

LLM에 근거를 먼저 찾아 붙여 답하게 함. 환각↓, 출처 있는 답.

여기선 질문→그래프 검색→근거→gpt-5.6 답변.

GraphRAG graph + RAG

RAG인데 검색 대상이 지식그래프. 관계 따라가며 근거 수집 → 다개념 질문에 강함.

여기선 벡터청크 대신 개념그래프를 검색.

Temporal 시계열 인식

각 사실에 시점을 붙여 "언제 참이었나" 추적. Graphiti가 temporal인 이유.

여기선 강의 순서(seq)를 시간축으로 매핑.

group_id 네임스페이스

하나의 DB 안에서 그래프를 과목별로 격리하는 칸막이.

여기선 group_id=project(딥러닝·최적화개론…). 안 섞임.

05 · 이 프로젝트 고유

SynapVox만의 장치

일반 GraphRAG 위에 이 앱이 얹은 것들.

교차연결 (cross-connection)

두 과목을 함께 볼 때, 정규화한 개념 이름이 정확히 같으면 프론트에서 파란 점선으로 이음. "딥러닝∩머신러닝=경사하강법" 같은 연결점. 백엔드 아닌 브라우저에서 계산.

그래프 시각화 ≠ 검색

화면의 그래프는 Neo4j를 Cypher로 직접 읽은 것. 검색·LLM 안 씀 → 빠르고 무료. 검색은 오직 질문할 때.

콜드 스타트

백엔드(Render 무료)가 유휴 시 잠들어 첫 요청에 최대 ~50초 깨는 시간. 그래서 처리 모달이 "서버 깨우는 중" 안내.

표시 이름 (SvxProject)

group_id는 ASCII만 → 한글 못 씀. 그래서 사람이 읽는 이름을 별도 노드에 저장해 /projects가 돌려줌(모든 기기 한글).

  3. RAG 기본

    LLM에 근거를 붙이는 이유(환각↓). → "RAG 개요" 입문 자료 1개.

    왜 청크 대신 그래프인가. → GraphRAG 개념 글 1–2개.

    API 두 개: add_episode·search + temporal·group_ids. → help.getzep.com/graphiti.

한 문장 요약

Graphiti는 Neo4j(창고) 위에서 OpenAI(두뇌)로 글을 개념그래프로 저장하고 하이브리드로 검색하는 프레임워크. 넣기=LLM추출→저장, 질문=검색→LLM답변 = GraphRAG. 그래프 그림 보기는 그냥 창고 조회일 뿐 RAG가 아니다.
