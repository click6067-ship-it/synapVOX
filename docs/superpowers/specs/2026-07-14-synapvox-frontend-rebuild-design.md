# SynapVox 프론트엔드 재구축 — 설계 문서

- 날짜: 2026-07-14
- 상태: 설계 승인 대기
- 대상 repo: `click6067-ship-it/synapVOX` (standalone). 백엔드(Graphiti)는 Render 유지.

## 1. 목표

standalone SynapVox 프론트엔드를 **처음부터 다시 만든다.** 팀 React 프론트의 기능은 전부 흡수하되 디자인은 폐기하고, **다글로(Daglo) 레퍼런스**를 따른 다크 테크 제품 UI로 재구축한다. 팀원에게 배포해 보여주는 것이 성공 기준.

**한 문장:** 노트·강의 텍스트를 붙여넣으면 개념이 세션을 가로질러 이어지는 **라이브 지식 그래프**를, 다글로처럼 입력창 중심 홈 + 좌측 드로워 + 다크 워크스페이스로 제공한다.

## 2. 범위

### In
- 좌측 세로 **드로워 네비** (로고·새 그래프·내 프로젝트·AI 질문·프로필)
- **홈**(히어로 없음, 다글로형): 헤드라인 + 중앙 텍스트 입력창(강의/노트 → 그래프 생성·추가) + 액션 타일 + 최근 프로젝트 카드 그리드
- **워크스페이스**(프로젝트 클릭 시): 3컬럼 = 소스(실제 강의 목록) · **라이브 포스 그래프**(애니메이션·탄성·드래그) · AI 채팅(RAG)
- **멀티 프로젝트**: 각 프로젝트 = Graphiti `group_id`. 홈 입력으로 새 프로젝트 생성/전환.
- 그래프 기능: 필터(개념근거/동시출현/다음세션/연속확장/핵심개념만), 노드 선택→상세, 검색, RAG 답변(근거 인용)
- 다크 테크 디자인 시스템(anti-slop), 반응형
- 배포: 프론트=Vercel, 백엔드=Render Graphiti(기존)

### Out (YAGNI / 다음 스프린트)
- STT/오디오 녹음(다글로의 녹음 타일은 SynapVox에선 텍스트 입력으로 대체)
- 회의 도메인 스키마(Decision/ActionItem 등) — 데이터는 딥러닝/교육 유지
- 인증/사용자 계정(공개 데모 키 유지)
- 협업/실시간 동기화

## 3. 아키텍처

```
[React+Vite SPA (Vercel)]
  ├─ 라우팅: 홈(/) · 워크스페이스(/p/:projectId)
  ├─ API 클라이언트(재시도 래퍼 — Render free-tier no-server 대응)
  └─ 라이브 포스 그래프(imperative rAF 시뮬레이션, 팀 프론트에서 만든 엔진 재사용·개선)
        │  fetch (CORS 개방)
        ▼
[Render Graphiti 백엔드 (gsvx, 기존)]
  ├─ 멀티 프로젝트: 엔드포인트가 project(group_id) 파라미터 수용 (신규)
  ├─ /projects (목록) · /graph · /ask · /ingest-text · /search · /config
  └─ Neo4j Aura (group_id로 프로젝트 격리)
```

- **프론트↔백 계약**: `X-API-Key`(공개 데모 키) + `project` 파라미터(group_id). 기존 key→project 고정 매핑을 project 파라미터로 일반화.
- **상태**: 라우팅(현재 프로젝트) + 프로젝트 목록 + 그래프 데이터 + 채팅 로그. 그래프 물리는 ref/rAF(재렌더 밖).

## 4. 화면 구조

### 4-1. 좌측 드로워 (persistent)
- 상단: SynapVox 로고(마크)
- 항목(아이콘+라벨): **새 그래프**(홈 입력 포커스) · **내 프로젝트**(목록) · **AI 질문**(현재 프로젝트 채팅) · (선택)**샘플**
- 하단: 프로필/설정. 접기(아이콘만) 토글. 모바일=오프캔버스.

### 4-2. 홈 `/` (다글로형, 히어로 없음)
- 헤드라인(중앙): 예 *"노트를 붙이면, 지식이 세션을 가로질러 이어진다"*
- **중앙 입력창**: 큰 텍스트 영역 "강의·노트를 붙여넣어 지식 그래프를 만드세요" + 제목 + 전송. **홈 입력 = 새 프로젝트 생성**(붙여넣은 텍스트가 첫 세션). 기존 프로젝트에 추가는 워크스페이스의 "텍스트로 추가"에서.
- **액션 타일 줄**: `텍스트로 추가` · `딥러닝 샘플 불러오기` · `그래프 보기` · `AI 질문` (다글로 타일 대응)
- **최근 프로젝트 카드 그리드**: 각 카드=프로젝트(제목·세션수·개념수·수정시각), 클릭→워크스페이스.

### 4-3. 워크스페이스 `/p/:projectId` (다크 3컬럼)
- **좌: 소스** — 실제 세션(강의) 목록(제목·개념수), 클릭→노드 하이라이트/상세. 상단 "텍스트로 추가" 재노출.
- **중앙: 라이브 그래프** — 애니메이션 포스(스프링 탄성·드래그·줌·팬), 필터 바(핵심개념만 포함), 범례. 노드 클릭→상세 패널.
- **우: AI 채팅** — 질문 입력→ `/ask` RAG 답변(근거 인용), 관련 노드 하이라이트.

## 5. 디자인 시스템 (다크 테크, anti-slop)

- **레퍼런스**: 다글로(다크 배경·좌측 아이콘 드로워·입력창 중심 메인·타일·최근 카드). 우리 각색 = 지식그래프 도메인.
- **팔레트(4-6색, daltonized 절제)**: 딥 다크 배경(#0e0f12류) + 표면(#16181d) + 텍스트(고대비 오프화이트/뮤트) + **시그니처 액센트 1**(그래프 개념=차분한 청록/에메랄드) + **액센트 2**(세션/관계=인디고). 보라 그라데이션·네온 남용 회피.
- **타이포 3종**: 디스플레이(헤드라인, Inter/Space Grotesk 회피 — 예: 산세 지오메트릭 1종) + 본문 산세 + **모노스페이스**(개념/코드 라벨=시그니처, 딥러닝 감성). 실제 폰트는 구현 시 확정.
- **시그니처 요소 1**: 살아있는 포스 그래프 자체(다크에서 발광하는 노드/엣지). 이게 제품의 얼굴.
- **spacing/radius**: 일관 리듬, 카드 radius 통일 회피(계층별 차등).
- **자기비판 게이트**: "이 화면이 아무 AI SaaS에나 낼 기본값처럼 읽히는가?" → 아니오가 될 때까지. 구현 후 `sloplint`(결정론) + `/vcheck` 병행.

## 6. 백엔드 변경 (멀티 프로젝트)

`gsvx/api.py`를 project 파라미터 수용으로 일반화:
- `GET /projects` (신규): 존재하는 group_id 목록 + 메타(세션수·개념수).
- `GET /graph?project=<id>` · `GET /ask?project=<id>&q=` · `POST /ingest-text {project, text, title}` · `GET /search?project=` : 고정 key_map 대신 project(group_id) 파라미터 사용. 미지정 시 기본 프로젝트.
- 인증은 기존 `DEMO_API_KEY` 게이트 유지. **홈 입력으로 프로젝트 생성하려면 쓰기 허용 필요 → 데모 기간 `SVX_READONLY=0`**(공개 쓰기, OpenAI 비용 노출 → 데모 후 되돌림·spend limit).
- Graphiti group_id 규칙: 영숫자·대시·언더스코어만(한글 불가) → 프로젝트 id는 slug.

## 7. 컴포넌트 분해 (isolation)

- `api/client.ts` — fetch+재시도 래퍼, project 파라미터. (무엇: 백엔드 호출. 의존: 없음)
- `graph/forceSim.ts` — 순수 물리(노드/엣지→위치). (무엇: 레이아웃/애니메이션. 의존: 없음)
- `graph/GraphCanvas.tsx` — SVG 렌더 + forceSim 구동 + 드래그/줌.
- `home/Home.tsx` · `home/CreateInput.tsx` · `home/ProjectGrid.tsx`
- `workspace/Workspace.tsx` · `workspace/SourcesPanel.tsx` · `workspace/ChatPanel.tsx`
- `nav/Drawer.tsx`
- `state/` — 프로젝트/그래프/채팅 스토어(경량, Context 또는 zustand 중 구현 시 결정).
각 유닛은 인터페이스로 소통, 독립 테스트 가능.

## 8. 배포 & 검증

- 프론트: Vercel(프로덕션 alias). 빌드 시 `VITE_API_BASE`=Render Graphiti.
- 백엔드: Render 재배포(멀티프로젝트 API).
- 검증: `tsc` 타입체크 · 헤드리스(playwright)로 홈→프로젝트 생성→그래프 애니메이션·드래그·RAG 동작 확인 · `sloplint`+`/vcheck` 디자인 게이트 · 콘솔에러 0.
