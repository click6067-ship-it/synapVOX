# SynapVox IA 재설계 기획서 — "Archive Dashboard + Graph Mode"

> Codex(기획·디자인 리드) 회의 결과 + 동결 백엔드 제약 반영. 구현 = Claude.
> **핵심 전환:** SynapVox는 *"그래프 앱"*이 아니라 **"강의를 지식 아카이브로 정리하고, 필요할 때 그래프로 분석하는 앱"** — 그래프는 홈이 아니라 **모드**.

---

## 0. 문제 & 방향
- **문제(사용자):** 프론트 구조화가 덜 됨 — 그래프가 앱 전체라 "정리/탐색 구조"가 약하고, 들어와도 뭘 할지 직관적이지 않음.
- **방향:** 다글로형 **구조화된 대시보드**를 기본 진입으로, 그래프 시각화는 사이드바 **버튼(모드)**으로. 프로젝트→단원→개념 계층. 시각화 스코프 3종(프로젝트별/전체/교차연결).

## 1. ✅ BLOCKER 결정 (사용자 승인 완료 2026-07-15)
1. **단원 = 세션 (확정).** 각 강의(세션)가 곧 단원. UI는 `1단원 · CNN`처럼 병기. 세션 위 별도 단원 그룹은 P2(프론트 수동 규칙).
2. **교차연결 = 정규화 label 정확일치만 (확정).** 개념 label 정규화(소문자·기호제거·조사제거) 후 정확히 같은 것끼리만 연결. **검증됨:** 딥러닝∩머신러닝 = 11개(경사하강법·과적합·손실함수·정규화·가중치·학습률…), 딥러닝∩생명과학 = 0(오탐 없음). stopword·숨김·confidence는 P1.

## 2. 정보구조 / 사이트맵
- **`/` = Archive Dashboard (기본 진입)** — 좌 사이드바 · 중앙 학습 대시보드 · (선택 시) 우측 상세/질문 패널.
- `/project/:project` — 특정 프로젝트 대시보드(딥러닝/생명과학/머신러닝).
- `/project/:project/session/:id` · `/project/:project/concept/:id` — 상세.
- `/graph?scope=project&project=…` · `?scope=all` · `?scope=cross` — 그래프 **모드**(분석용).
- 흐름: **대시보드에서 "강의 추가 → 세션 확인 → 개념 탐색 → 질문", 필요할 때 그래프로.**

## 3. 다글로 벤치마크 (레이아웃/IA만, 색·타이포는 Archive 유지)
- **취할 것:** 좌 사이드바(영역·프로젝트 접근) · 중앙 상단 입력/CTA("강의 붙여넣기") · 빠른 액션(강의추가·질문·그래프) · 최근 작업/보드(최근 세션·개념·프로젝트) · 빈 상태에 "첫 루틴" 제시.
- **버릴 것:** 다글로식 범용 AI 기능 격자(우린 지식 구조화 앱), 모델 선택 드롭다운(백엔드에 없음), 과한 기능 런처.
- 근거: daglo.ai 워크스페이스("업로드→전사→구조화→저장"), 메인이 "한눈에 보고 선택" 구조.

## 4. 메인 대시보드 구성
1. **Top Workbench** — 제목 `오늘 정리할 강의` · 1차 CTA `강의 추가` · 보조 `질문하기` · 빈 상태 `강의 텍스트를 붙여넣으면 단원과 개념 지도가 생깁니다.` + `첫 강의 추가` 버튼.
2. **Project Shelf** — 프로젝트 카드(딥러닝·생명과학·머신러닝), sessions/concepts 수, 최근 세션 2개 미리보기, `열기`·`그래프 보기`.
3. **Outline View** — 프로젝트→세션(단원)→개념 **아코디언 아웃라인**. 예:
   - 생명과학 → 1단원 세포(세포막·ATP·효소) / 2단원 염색체(DNA·유전자·감수분열)
   - 딥러닝 → 1 CNN(convolution·filter·pooling) / 2 RNN(hidden state·sequence·gradient)
4. **Question Dock** — `이 프로젝트에 질문하기`, 답변 후 근거 세션 클릭→상세, `그래프에서 근거 보기`→그래프 모드 deep-link.

## 5. 사이드바
- 상단: `SynapVox Archive` + 프로젝트 스위처.
- **Primary:** `＋강의 추가` · `대시보드` · `질문하기` · **`그래프 시각화`**.
- Projects: 딥러닝 · 생명과학 · 머신러닝 · `모든 과목`.
- **Graph Scope**(그래프 버튼 시 노출): segmented `현재 프로젝트 / 전체 / 교차연결`.
- Archive: 최근 세션 · 핵심 개념 · stats.
- 그래프 버튼: 대시보드에선 Primary 고정 → 클릭 시 `/graph`. 현재 프로젝트면 scope=project, 모든 과목이면 scope=all, **교차연결은 오탐 가능성 있어 기본값 아님(명시 선택).**

## 6. 그래프 모드 (기존 Archive Graph Studio 강점 보존)
- **프로젝트별:** `/graph?project` — 계층색 노드(메인 아이보리 채움 / 세션 버밀리언·개념 라임(핵심)·틸(일반) 속빈), 우측 inspector.
- **전체(은하):** 프로젝트별 `/graph` 병합 + 허브 멀리(mainRepel), 프로젝트 라벨/링.
- **교차연결:** 개념 label 정규화 → 2+ 프로젝트 공유 label에 synthetic cross node(작은 아이보리 ring+blue halo) + **교차 edge = rule-blue 점선**, inspector에 `프론트 추정 연결` 표기. 노이즈: P0 정확일치 / P1 stopword·토글 / P2 사용자 숨김.

## 7. 첫 진입 직관성 (3초 안에 "뭘 하면 되는지")
- **빈 상태:** 중앙 단일 CTA `첫 강의 추가` + 3단계(`1.강의 붙여넣기` `2.단원·개념 자동 정리` `3.질문/그래프로 보기`) + `샘플로 보기: 생명과학 / 딥러닝 / 머신러닝`.
- **데이터 있을 때:** 최상단 "최근 이어서 볼 단원", 프로젝트 카드는 stats보다 outline 미리보기 우선, 그래프는 보이되 1차 행동 아님.

## 8. 샘플 프로젝트 (요청)
- **딥러닝(P-BIO)** 7세션 · **생명과학(P-LIFE)** · **머신러닝(P-ML, 신규)** — ML은 딥러닝과 개념 공유(경사하강법·과적합·손실함수)라 교차연결 데모에 적합. projectLabel 매핑(P-BIO→딥러닝 등).

## 9. 우선순위
- **P0 구조 전환:** 대시보드 기본 진입 · 사이드바 `그래프 시각화` · 프로젝트→세션(단원)→개념 아웃라인 · 빈 상태 온보딩 · 그래프 scope(현재/전체) · Archive 토큰 유지.
- **P1 교차연결:** `cross` scope · label 정규화 정확일치 · synthetic cross node/점선 · inspector "추정 연결" · stopword.
- **P2 학습 UX:** 세션 수동 grouping(프론트 규칙) · 교차 confidence·숨김·pin · 질문↔아웃라인↔그래프 양방향 · 라벨 매핑 UI.

## 10. 재사용 / 구현
- 재사용: 기존 GraphView(계층색·은하·mainRepel·탄성)·api client·mapGraph·useAsk/useDetail·토큰. 신규: 대시보드 화면·아웃라인·라우팅·사이드바 개편·교차연결 계산.
